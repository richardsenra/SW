Guia de Utilização

📋 Qual a proposta da ferramenta?

A proposta é permitir que qualquer usuário, mesmo sem conhecimento técnico, possa iniciar um processo de negócio (workflow) do Bitrix24 para uma grande quantidade de itens (Leads, Negócios, etc.) de uma só vez, a partir de uma simples planilha.

Ela foi criada para ser segura, evitando os erros mais comuns de sobrecarga da API do Bitrix24 e fornecendo logs detalhados para auditoria e depuração.

🔧 Explicando os Campos

Seu Nome

Usado para registrar quem realizou a operação. Aparecerá na primeira linha dos logs.

URL do Webhook

É a "ponte" de comunicação com a sua conta Bitrix24. Deve ter, no mínimo, a permissão de "Workflows (bizproc)".

ID do Template de Workflow

É o número de identificação do processo de negócio que você quer executar para cada item da lista.

Tipo da Entidade

O módulo do Bitrix24 onde o workflow será executado (Negócios, Leads, Contatos ou Empresas).

Arquivo com IDs

Onde você carrega sua planilha (.xlsx ou .csv) contendo a lista de IDs.

Nome da Coluna com IDs

O nome exato da coluna na sua planilha onde estão os IDs (o padrão é "ID").

Pular os Primeiros N Itens

Permite continuar um trabalho que parou. Se um processo rodou 500 itens e deu erro, você pode colocar "500" aqui.

Tamanho do Lote vs. Pausa

Os dois controles de carga mais importantes para evitar sobrecarga da API.

⚙️ Entendendo a Configuração de Carga

O Bitrix24 tem limites de processamento. Se você enviar muitas tarefas ao mesmo tempo, ele pode recusar ou falhar. Pense nisso como um caixa de supermercado:

  • Tamanho do Lote: É a quantidade de produtos que você coloca na esteira de uma vez.
  • Pausa (sleep): É o tempo que você espera antes de colocar um novo grupo de produtos na esteira.

O objetivo é encontrar o equilíbrio para não sobrecarregar o caixa (o servidor do Bitrix24).

📊 Categorias de Workflow e Configurações

🟢 Workflow BÁSICO

O que é: Um processo que realiza poucas ações e que não depende de serviços externos.

Exemplos:
  • Mudar o estágio de um card
  • Alterar o nome de um negócio/lead
  • Adicionar um comentário simples
  • Preencher um ou dois campos
Configuração Recomendada:
Tamanho do Lote: 40 a 50
Pausa (segundos): 0.5 a 1.0

🟡 Workflow MÉDIO

O que é: Um processo que envolve algumas ações, lógica condicional ou uma única chamada para um webhook externo.

Exemplos:
  • Criar uma ou duas tarefas para um usuário
  • Enviar uma notificação
  • Disparar um webhook para um fluxo simples no n8n
  • Modificar vários campos e adicionar comentários
Configuração Recomendada:
Tamanho do Lote: 20 a 30
Pausa (segundos): 1.5 a 2.0

🔴 Workflow COMPLEXO

O que é: Um processo longo, com muitas etapas, pausas, ou que depende de várias chamadas a sistemas externos.

Exemplos:
  • Workflows que chamam n8n/Make que consultam outras APIs
  • Processos com pausas internas ("wait")
  • Processos que criam várias entidades
  • Loops ou iterações dentro do workflow
Configuração Recomendada (mais segura):
Tamanho do Lote: 5 a 15
Pausa (segundos): 2.5 ou mais

💡 Conclusão

Na dúvida, comece sempre pela configuração mais segura (a de workflow complexo). É melhor que o processo demore um pouco mais e termine com sucesso do que ele parar no meio por um erro de sobrecarga. Use os logs para monitorar o comportamento e otimize os parâmetros conforme se sentir confortável.

Processo Interrompido

Ocorreu um erro crítico de 'OPERATION_TIME_LIMIT'. O processo foi parado para evitar falhas. Verifique os logs, ajuste o 'Tamanho do Lote' para um valor menor e tente novamente.

Permissão Insuficiente

O webhook não tem a permissão 'Workflows' (bizproc). Por favor, edite o webhook no Bitrix24, adicione essa permissão e tente novamente.
